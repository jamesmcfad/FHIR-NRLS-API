The 'delete' interaction removes an existing resource. The interaction is performed by an HTTP DELETE of the {{include.resource}} resource.

All requests SHALL contain a valid ‘Authorization’ header and SHALL contain an ‘Accept’ header. The `Accept` header indicates the format of the response the client is able to understand, this will be one of the following application/json+fhir or application/xml+fhir.

1.1. Response

Assuming successful URL submission, there are two possible outcomes to the delete request:

A full set of response codes can be found here API Response Codes. FHIR Servers SHALL support the following response codes:

204 No Content The record either does not exist or has been successfully deleted.
404 No match to delete The parameters used to identify the record were not met.
